# List App - 轻量级列表聚合工具

**版本：** 0.1-draft  
**日期：** 2026-03-01  
**作者：** @Fizzzli

---

## 产品定位

**一句话：** 一个 App 记录所有类型的列表，轻量如纸笔，分享如 Git。

**目标用户：** 
- 喜欢记录生活的人（演出、餐厅、书单等）
- 厌倦了用多个 App 管理不同列表
- 想要快速记录，不被复杂功能打扰

**核心价值：**
1. **聚合** - 所有列表在一个地方
2. **轻量** - 3 秒内完成一条记录
3. **分享** - 列表可以 Fork，像 GitHub 一样

---

## 核心功能

### 1. 列表类型

**预置模板：**
| 类型 | 字段 |
|------|------|
| 🎵 Live 演出 | 艺人、场地、日期、票务状态、评分 |
| 🍜 餐厅 | 店名、菜系、地址、想去/去过、评分 |
| 📚 书单 | 书名、作者、状态、评分、笔记 |
| 🎬 影单 | 片名、导演、平台、状态、评分 |
| ✈️ 旅行清单 | 目的地、日期、待办事项、打包清单 |

**自定义类型：** 用户可创建自己的模板，定义字段。

### 2. 条目操作

- ✅ 快速添加（名称 + 日期即可）
- ✅ 状态流转（待办 → 进行中 → 完成）
- ✅ 备注/评分/图片
- ✅ 批量操作（多选标记完成）

### 3. 分享与 Fork

**分享模式：**
- 公开列表 → 生成链接/二维码
- 私密列表 → 仅自己可见

**Fork 机制：**
- 看到别人的列表 → 一键克隆到自己账户
- Fork 后完全独立，可自由修改
- 可选择是否关联原列表（接收更新通知）

### 4. 离线优先

- 无网络也能完整使用
- 数据本地存储
- 联网后自动同步（如果开启云同步）

## 数据模型

### 核心 Entity

```kotlin
// 列表定义
data class ListTemplate(
    val id: String,
    val name: String,
    val icon: String,
    val fields: List<FieldDefinition>,
    val isSystem: Boolean,  // 预置模板不可删除
    val createdAt: Long,
    val ownerId: String?    // null = 系统预置
)

data class FieldDefinition(
    val key: String,
    val label: String,
    val type: FieldType,    // TEXT, DATE, NUMBER, SELECT, IMAGE
    val required: Boolean,
    val options: List<String>?  // SELECT 类型的选项
)

// 用户创建的列表实例
data class UserList(
    val id: String,
    val templateId: String,
    val title: String,
    val isPublic: Boolean,
    val forkedFrom: String?,  // 如果是 fork 的，记录原列表 ID
    val createdAt: Long,
    val updatedAt: Long,
    val ownerId: String
)

// 列表条目
data class ListItem(
    val id: String,
    val listId: String,
    val name: String,
    val status: ItemStatus,  // PENDING, IN_PROGRESS, COMPLETED
    val fields: Map<String, String>,  // 动态字段值
    val createdAt: Long,
    val completedAt: Long?
)

enum class ItemStatus {
    PENDING,      // 想去/待办
    IN_PROGRESS,  // 已买票/进行中
    COMPLETED     // 已完成/去过
}
```

### Room Database Schema

```
ListTemplate (1) ──< UserList (1) ──< ListItem
     │                    │
     └──── FieldDefinition ┘
```

## 技术栈

### 前端（Android）

| 组件 | 选型 | 理由 |
|------|------|------|
| 语言 | Kotlin | 官方推荐 |
| UI | Jetpack Compose | 现代、声明式、开发快 |
| 数据库 | Room | 官方 ORM，离线优先 |
| 配置 | DataStore | 替代 SharedPreferences |
| 依赖注入 | Hilt | 官方推荐 |
| 图片加载 | Coil | Kotlin 优先，轻量 |

### 后端（可选，Phase 3）

**方案 A：Firebase（推荐 MVP）**
- Authentication：匿名登录 + 邮箱/Google
- Firestore：列表数据同步
- Storage：图片存储
- 免费额度够用，开发最快

**方案 B：Supabase**
- 开源、可自托管
- PostgreSQL + Realtime
- 便宜，但需要更多配置

**方案 C：自建**
- Spring Boot / Ktor
- 完全可控
- 维护成本高

**MVP 建议：** 纯本地版（Phase 1-2），验证需求后再加后端。
